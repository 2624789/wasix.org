## Non-Invasive extensions added to WASI

### WASIX extended Functions

- The ability to duplicate file handles
  - [`fd_dup`](/docs/api-reference/wasix/fd_dup)
- polling functions that can be interrupted by libraries such as `tokio` and `mio`
  - [`fd_event`](/docs/api-reference/wasix/fd_event)
- Pipes to stream data to and from subprocesses
  - [`fd_pipe`](/docs/api-reference/wasix/fd_pipe)
- Ability to yield CPU time without the bloat of `poll_oneoff`
  - `sched_yield`
- Getting and setting the TTY properties
  - [`tty_get`](/docs/api-reference/wasix/tty_get)
  - [`tty_set`](/docs/api-reference/wasix/tty_set)
- Changing the current directory is now natively supported
  - [`getcwd`](/docs/api-reference/wasix/getcwd)
  - [`chdir`](/docs/api-reference/wasix/chdir)
- Signals can be blocked (needed by `libc`)
  - [`callback_signal`](/docs/api-reference/wasix/callback_signal)
- Spawning threads as per the experimental threads spec
  - [`thread_spawn`](/docs/api-reference/wasix/thread_spawn)
- Extra thread functions
  - [`thread_sleep`](/docs/api-reference/wasix/thread_sleep)
  - [`thread_id`](/docs/api-reference/wasix/thread_id)
  - [`thread_join`](/docs/api-reference/wasix/thread_join)
  - [`thread_parallelism`](/docs/api-reference/wasix/thread_parallelism)
  - [`thread_signal`](/docs/api-reference/wasix/thread_signal)
  - [`thread_exit`](/docs/api-reference/wasix/thread_exit)
- Operating system futex support used for multithread constructs
  - [`futex_wait`](/docs/api-reference/wasix/futex_wait)
  - [`futex_wake`](/docs/api-reference/wasix/futex_wake)
  - [`futex_wake_all`](/docs/api-reference/wasix/futex_wake_all)
- Longjmp and setjmp used by `libc`
  - [`stack_checkpoint`](/docs/api-reference/wasix/stack_checkpoint)
  - [`stack_restore`](/docs/api-reference/wasix/stack_restore)
- Subprocess support
  - [`proc_raise_interval`](/docs/api-reference/wasix/proc_raise_interval)
  - [`proc_fork`](/docs/api-reference/wasix/proc_fork)
  - [`proc_exec`](/docs/api-reference/wasix/proc_exec)
  - [`proc_spawn`](/docs/api-reference/wasix/proc_spawn)
  - [`proc_id`](/docs/api-reference/wasix/proc_id)
  - [`proc_parent`](/docs/api-reference/wasix/proc_parent)
  - [`proc_join`](/docs/api-reference/wasix/proc_join)
  - [`proc_signal`](/docs/api-reference/wasix/proc_signal)
- Interface support
  - [`port_bridge`](/docs/api-reference/wasix/port_bridge)
  - [`port_unbridge`](/docs/api-reference/wasix/port_unbridge)
  - [`port_dhcp_acquire`](/docs/api-reference/wasix/port_dhcp_acquire)
  - [`port_addr_add`](/docs/api-reference/wasix/port_addr_add)
  - [`port_addr_remove`](/docs/api-reference/wasix/port_addr_remove)
  - [`port_addr_clear`](/docs/api-reference/wasix/port_addr_clear)
  - [`port_mac`](/docs/api-reference/wasix/port_mac)
  - [`port_addr_list`](/docs/api-reference/wasix/port_addr_list)
  - [`port_gateway_set`](/docs/api-reference/wasix/port_gateway_set)
  - [`port_route_add`](/docs/api-reference/wasix/port_route_add)
  - [`port_route_remove`](/docs/api-reference/wasix/port_route_remove)
  - [`port_route_clear`](/docs/api-reference/wasix/port_route_clear)
  - [`port_route_list`](/docs/api-reference/wasix/port_route_list)
- All the missing socket functionality
  - [`sock_status`](/docs/api-reference/wasix/sock_status)
  - [`sock_addr_local`](/docs/api-reference/wasix/sock_addr_local)
